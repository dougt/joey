Make sure that the uploads directory are writable by the Apache server.

The login.php script authenticate the user for the session. Use this form to test this and authenticate your session.

 
POST URL: 
  login.php 
POST params: 
  username -- the uname column in the DB
  password

The upload.php script uploads an entry to the server. Use this form to test this. You must be logged in first.

 
POST URL: 
  upload.php 
POST params: 
  name
  title
  uri
  size
  uuid
  type -- MIME type audio / video / image are saved in files
  data -- bookmark, microsummary, or base64 encoded medis file data

The logout.php script logs out of the current session. It does not take any POST parameters. Click here to run it.

The count.php script returns the number of upload entries for the current logged in user. It does not take any POST parameters. Click here to run it.

The getdata.php script retrieves entries for this user from the database. It returns "-1" if the user is not authenticated. Use this form to try it.

POST URL: 
  getdata.php 
POST params: 
  first -- first entry from the list
  pagesize -- number of entries returned
  sendData -- whether to send data with the entry (or just file names)

Depending on whether the sendData parameter is set. The script returns the following data in the HTTP response body. You need to use "view source" in your browser to view the data.

$id\n
$name\n
$uri\n
$title\n
$date_created\n
$type\n
$content\n
$filename\n      OR [base64 encoded file data]\n
$thumbnailname\n OR [base64 encoded file data]\n
